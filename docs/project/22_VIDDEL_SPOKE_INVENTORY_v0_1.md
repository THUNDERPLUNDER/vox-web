# Viddel Spoke Inventory v0.1

## Status

Strukturert oversikt med **12 innholdskandidater** for Viddel MVP. Dokumentet er **arbeidsgrunnlag** for videre artikkelproduksjon og for senere hub- og forside-design — ikke en roadmap med tidslinje eller leveransefaser.

**Skrive- og brief-standard:** `docs/project/21_VIDDEL_ARTICLE_BRIEF_v0_1.md`

## Hub-modell (første arbeidsversjon)

| Hub | Fokus i v0.1 |
|-----|----------------|
| Bedre lyd i hverdagen | Hverdagsbelastning, sosiale situasjoner, når å søke profesjonell hjelp |
| Feilsøking og teknikk | Tilkobling, apparat-/lydopplevelse, app, filter og enkel vedlikehold |
| Ny med høreapparat | Oppstart, forventninger, «hva er normalt» |
| Pårørende og hjelpere | Støtte i samtaler uten å ta over |
| Restart og mestring | Lav bruk, motstand, tilbake etter pause |

## Om titler

Hver kandidat har **anbefalt arbeidstittel**, **alternativ** og **begrunnelse**. Titler er varierte (problem + handling, «Slik …», «Dette er normalt …», «Før du …», «Prøv dette …»); **«Når …»** brukes bare der det gir tydelig beslutnings- eller situasjonsanker.

## Kandidat 01 — P1

| | |
|---|---|
| **Anbefalt arbeidstittel** | Koble høreapparatet til mobilen: steg som løser det meste |
| **Alternativ tittel** | Når telefonen ikke finner høreapparatet |
| **Hvorfor anbefalt** | Legger **konkret handlingsløype** først; alternativ beholder kjent problemformulering for søk og chat-seed. |

**Hub:** Feilsøking og teknikk  
**Artikkeltype:** Spoke (praktisk guide)

**Brukersituasjon:** Vil ha stabil lyd eller tilkobling mellom høreapparat og mobil.  
**Problem brukeren prøver å løse:** Bluetooth kobler ikke, faller ut, eller står som tilkoblet uten lyd.  
**Hva brukeren skal forstå:** De fleste feil følger gjenkjennelige mønstre (sesjon, tillatelser, nærhet/støy).  
**Hva brukeren skal gjøre:** Gå gjennom en **kort, prioritert** sjekk før eskalering.  
**Første praktiske steg:** Slå av/på begge enheter; fjern apparat fra Bluetooth-listen og parkoble på nytt.  
**Når menneskelig hjelp bør anbefales:** Feilen består etter full sjekk; kun lyd på én side; væske/fall; smerte.

**AI-seedspørsmål (3):**

1. Hva gjør jeg først når høreapparatet ikke kobler til telefonen?  
2. Bluetooth sier tilkoblet men jeg hører ikke — hva betyr det?  
3. Hvor finner jeg riktig app til mitt merke?

**RAG-/kildebehov:** Merke-spesifikke app-navn og anbefalt parringsrekkefølge (prod./support); generell Bluetooth-feilsøking.  
**Visuelt anker:** Tallrekke 1–4 (telefon, høreapparat, app, Bluetooth); unngå merke-spesifikk skjermdump i generisk mal.

## Kandidat 02 — P1

| | |
|---|---|
| **Anbefalt arbeidstittel** | Slutt med pipen: vanlige årsaker og raske grep |
| **Alternativ tittel** | Når høreapparatet piper |
| **Hvorfor anbefalt** | Lovet **utfall** (mindre pip); alternativ matcher daglig talemål. |

**Hub:** Feilsøking og teknikk  
**Artikkeltype:** Spoke (årsak + handling)

**Brukersituasjon:** Plagsomt eller hyppig pip fra apparatet.  
**Problem brukeren prøver å løse:** Finne årsak (batteri, sitte, feedback, voks) og dempe eller fjerne pip.  
**Hva brukeren skal forstå:** Pip er ofte **signal om noe konkret**, ikke «defekt uten grunn».  
**Hva brukeren skal gjøre:** Systematisk kort sjekk (batteri/lad, domes, ørekanal, enkel trygg reset der det beskrives).  
**Første praktiske steg:** Bytt batteri eller bekreft ladestatus; sjekk at domes/propp sitter riktig.  
**Når menneskelig hjelp bør anbefales:** Smerte, væske etter bad, fall; pip stopper ikke etter trygg sjekk.

**AI-seedspørsmål (3):**

1. Hvorfor piper høreapparatet mitt hele tiden?  
2. Kan jeg gjøre noe selv før jeg kontakter audiograf?  
3. Hva er forskjellen på pip fra apparat og piping i øret som ikke er apparatet?

**RAG-/kildebehov:** Prod.-FAQ om pip og feedback; generell hygienisk sikkerhet; **ikke** medisinsk diagnose av øresykdom.  
**Visuelt anker:** Tre korte «årsak → én handling»-kort.

## Kandidat 03 — P1

| | |
|---|---|
| **Anbefalt arbeidstittel** | De første ukene med høreapparat: dette er normalt |
| **Alternativ tittel** | Nytt høreapparat: forventningsguide for oppstart |
| **Hvorfor anbefalt** | Følger brief-mønster **normalisering + neste steg**; god for premature dropout. |

**Hub:** Ny med høreapparat  
**Artikkeltype:** Explainer med handlingsdel

**Brukersituasjon:** Nylig oppstart; usikker på uvanlig lyd, tretthet og plagsomhet.  
**Problem brukeren prøver å løse:** Skille **tilvenning** fra tegn på at noe er galt.  
**Hva brukeren skal forstå:** Hjernen trenger repetering; korte, regelmessige økter beats «alt eller ingenting».  
**Hva brukeren skal gjøre:** Plan for daglig bruk i trygge omgivelser + notere **ett** problem per uke til audiograf.  
**Første praktiske steg:** Bruk apparatet hjemme i rolig miljø 1–2 timer (juster etter egen audiografs råd).  
**Når menneskelig hjelp bør anbefales:** Sterk smerte, sår hud, akutt hørselstap, uttalt psykisk belastning.

**AI-seedspørsmål (3):**

1. Er det normalt at alt høres annerledes de første dagene?  
2. Hvor mye bør jeg bruke apparatet den første uka?  
3. Når skal jeg booke ny time hos audiograf i oppstartsfasen?

**RAG-/kildebehov:** Pasientorientert materiale om tilvenning; ikke erstatte individuell JBL-plan.  
**Visuelt anker:** Enkel tidslinje uke 1–4 med 3–4 milepæler (ikke medisinsk presisjonstall i UI).

## Kandidat 04 — P2

| | |
|---|---|
| **Anbefalt arbeidstittel** | Prøv dette når lyden blir slitsom utpå dagen |
| **Alternativ tittel** | Hørselstretthet: når ørene trenger pause |
| **Hvorfor anbefalt** | **Handlingsnær** åpning; alternativ introduserer begrep før grep. |

**Hub:** Bedre lyd i hverdagen  
**Artikkeltype:** Spoke (hverdagsmestring)

**Brukersituasjon:** Etter flere timer med samtale/støy blir lytting tungt.  
**Problem brukeren prøver å løse:** Orke dagen uten å isolere seg helt.  
**Hva brukeren skal forstå:** Lyttebelastning hopper ofte gradvis; miljøgrep før tunge innstillings-endringer.  
**Hva brukeren skal gjøre:** Mikropauer, bedre plass i rom, tydelig turtaking hos andre, evt. programvalg på apparat om tilgjengelig.  
**Første praktiske steg:** 5–10 minutter med redusert lydbelastning (stille rom / dempet program) etter behov.  
**Når menneskelig hjelp bør anbefales:** Symptomer som hindrer jobb/søvn, eller plutselig hørselstap.

**AI-seedspørsmål (3):**

1. Hvorfor blir samtaler ekstra tunge utpå ettermiddagen?  
2. Hva kan jeg gjøre på jobben når jeg blir utmattet av å lytte?  
3. Hvordan skille lyttebelastning fra at apparatet er feil innstilt?

**RAG-/kildebehov:** Lyttebelastning og kognitive effekter (fagforklaring lett tilgjengelig); skill fra generell utbrenthet.  
**Visuelt anker:** Stilisert «belastning over dagen» (kurve eller enkel grafikk, ikke klinisk).

## Kandidat 05 — P2

| | |
|---|---|
| **Anbefalt arbeidstittel** | Slik støtter du uten å ta over samtalen |
| **Alternativ tittel** | Hjelpe noen som hører dårlig — uten å kapre ordet |
| **Hvorfor anbefalt** | Tydelig **atferdsmål** for pårørende; «Slik …» er standard bruksoppskrift. |

**Hub:** Pårørende og hjelpere  
**Artikkeltype:** Spoke / veileder

**Brukersituasjon:** Partner, voksen barn eller venn vil støtte i gruppe eller kalender.  
**Problem brukeren prøver å løse:** Unngå oversnakk, irritasjon og tap av autonomi for den som hører.  
**Hva brukeren skal forstå:** Korte reformuleringer og turtaking hjelper mer enn å «administrere» samtalen.  
**Hva brukeren skal gjøre:** Praktisere ett nytt grep per samtale (vent litt, snu mot, korte setninger).  
**Første praktiske steg:** Avtal et tegn for «gjenta» som ikke er nedsettende.  
**Når menneskelig hjelp bør anbefales:** Vedvarende konflikt; behov for familievernkontakt (nøytral henvisning).

**AI-seedspørsmål (3):**

1. Hvordan kan jeg gjenta uten at det føles nedsettende?  
2. Hva gjør jeg i støyende restaurant som pårørende?  
3. Hvordan be andre om tydelig tale uten å lage «scene»?

**RAG-/kildebehov:** Kommunikasjon ved hørselstap (pasienthåndbøker, vel ansett folkehelsemateriale).  
**Visuelt anker:** To kolonner «prøv / unngå» med korte eksempler.

## Kandidat 06 — P2

| | |
|---|---|
| **Anbefalt arbeidstittel** | Appen finner ikke høreapparatet: sjekkliste du kan gå nå |
| **Alternativ tittel** | Når appen ikke finner høreapparatet |
| **Hvorfor anbefalt** | **Sjekkliste** matcher Google/søk-intent og chat; «Når …» som ALT. |

**Hub:** Feilsøking og teknikk  
**Artikkeltype:** Spoke (teknisk)

**Brukersituasjon:** App viser ingen enheter eller søker uten treff.  
**Problem brukeren prøver å løse:** Gjenopprette kontakt app ↔ apparat.  
**Hva brukeren skal forstå:** Tillatelser, nærhet, riktig app og ofte «hard» omstart av kjeden.  
**Hva brukeren skal gjøre:** Gå radert sjekk: app-versjon, BT, fjerne parring, logge inn på nytt der relevant.  
**Første praktiske steg:** Lukk app helt (force close), slå Bluetooth av/på, åpne app på nytt nær apparatet.  
**Når menneskelig hjelp bør anbefales:** Feil etter OS-oppdatering på tvers av enheter; mistanke om defekt.  

**AI-seedspørsmål (3):**

1. Appen ser ikke høreapparatet — første steg?  
2. Trenger jeg ny telefon for at appen skal fungere?  
3. Hva er parringsmodus på høreapparatet, og når bruker jeg det?

**RAG-/kildebehov:** Merke-app dokumentasjon; iOS/Android generelle Bluetooth-anbefalinger.  
**Visuelt anker:** Numret sjekkliste (tilgjengelig semantikk, evt. avkryssing).

## Kandidat 07 — P3

| | |
|---|---|
| **Anbefalt arbeidstittel** | Bare lyd på én øre: før du ringer audiograf |
| **Alternativ tittel** | Når høreapparatet bare virker på én side |
| **Hvorfor anbefalt** | **«Før du …»** setter forventning og trygge første grep; alternativ er ren situasjonsbeskrivelse. |

**Hub:** Feilsøking og teknikk  
**Artikkeltype:** Spoke

**Brukersituasjon:** Lyd eller effekt kun på ett øre eller én enhet i par.  
**Problem brukeren prøver å løse:** Avklare om det er ørekanal, innstilling eller enhet.  
**Hva brukeren skal forstå:** Mono/stereo og dual-stream feil har ulike «første sjekker».  
**Hva brukeren skal gjøre:** Bytt domes side, sjekk program, restart begge enheter, bytt batteri på berørt enhet.  
**Første praktiske steg:** Bekreft at begge enheter er på og sitter som vanlig.  
**Når menneskelig hjelp bør anbefales:** Plutselig asymmetrisk hørsel; smerte; ingen endring etter trygg sjekk.

**AI-seedspørsmål (3):**

1. Jeg hører bare på ett øre med høreapparat — hva betyr det?  
2. Kan det være feil program når bare en side virker?  
3. Når må jeg til audiograf med én‑sides problem?

**RAG-/kildebehov:** Prod.-dok om binaural kobling og streaming; audiologisk eskalering.  
**Visuelt anker:** Enkel illustrasjon «to enheter / ett signal».

## Kandidat 08 — P3

| | |
|---|---|
| **Anbefalt arbeidstittel** | Bytte filter uten stress: en enkel rekkefølge |
| **Alternativ tittel** | Slik bytter du filter uten å bli usikker |
| **Hvorfor anbefalt** | **Prosess** og trygghet i tittelen; alternativ er uttrykkelig «Slik …». |

**Hub:** Feilsøking og teknikk  
**Artikkeltype:** Spoke (prosedyre)

**Brukersituasjon:** Voks/propp/filter skal byttes; redd for å ødelegge noe.  
**Problem brukeren prøver å løse:** Fullføre bytte rent og trygt.  
**Hva brukeren skal forstå:** Feil filtertype gir dårlig lyd eller feedback; riktig variant er billig forsikring.  
**Hva brukeren skal gjøre:** Følg merke/modell-spesifikk rekkefølge; ha reserve på lur.  
**Første praktiske steg:** Vask hender, godt lys, legg ren overflate.  
**Når menneskelig hjelp bør anbefales:** Kan ikke få på plass del; knekket receiver-ledning; kraftig feedback etter bytte.

**AI-seedspørsmål (3):**

1. Hvor ofte bør jeg bytte voksfilteret?  
2. Hva er tegn på at filteret er tett?  
3. Hvor finner jeg riktig reservedel til mitt merke?

**RAG-/kildebehov:** Merke-spesifikke byttevideoer/ID av delenummer; generell hygiene.  
**Visuelt anker:** Trinn 1–N med foto/illus av **generisk** filtertype (eller merkeplassholder).

## Kandidat 09 — P3

| | |
|---|---|
| **Anbefalt arbeidstittel** | Stemmen din høres annerledes ut: hva som ofte ligger bak |
| **Alternativ tittel** | Når stemmen din høres rar ut i høreapparatet |
| **Hvorfor anbefalt** | **Forklarer** uten alarm; unngår «Når …» som åpningsord. |

**Hub:** Ny med høreapparat  
**Artikkeltype:** Explainer kort + grep

**Brukersituasjon:** Egen stemme høres «metallisk», boksete eller fremmed.  
**Problem brukeren prøver å løse:** Forstå om det er tilvenning eller innstilling.  
**Hva brukeren skal forstå:** Occlusion, åpen/lukket oppsett og tilvenning er vanlige forklaringer.  
**Hva brukeren skal gjøre:** Notere når det er verst; teste enkle program eller domes-justering **med** audiografplan.  
**Første praktiske steg:** Snakk høyt i rolig rom 5 minutter daglig for tilvenning (hvis ikke smerte).  
**Når menneskelig hjelp bør anbefales:** Smerte, trykk, eller lukt/flyt fra øre.

**AI-seedspørsmål (3):**

1. Hvorfor hører jeg meg selv så annerledes med apparat?  
2. Hvor lenge varer det før egen stemme føles naturlig?  
3. Kan domes- eller ørestykkevalg fikse metallisk lyd?

**RAG-/kildebehov:** Audiolog forklaring på occlusion og egeneffekt; grense mot medisinsk.  
**Visuelt anker:** Enkel side‑ved‑side «før/etter» **konsept** (ikke lydløfte-markedsføring).

## Kandidat 10 — P3

| | |
|---|---|
| **Anbefalt arbeidstittel** | Før restaurant eller stort selskap: små grep som hjelper |
| **Alternativ tittel** | Når du gruer deg til restaurant eller familieselskap |
| **Hvorfor anbefalt** | **Forberedelse** og trygghet; «Når du gruer deg …» fungerer som sensitiv ALT. |

**Hub:** Bedre lyd i hverdagen  
**Artikkeltype:** Spoke (situasjonsplan)

**Brukersituasjon:** Støy, mange stemmer, prestasjonsangst rundt samtale.  
**Problem brukeren prøver å løse:** Redusere sosialt stress uten å avlyse.  
**Hva brukeren skal forstå:** Plass, akustikk og pauser teller like mye som «perfekt hørsel».  
**Hva brukeren skal gjøre:** Reservere sete, forberede én setning å si ifra med, bruke program for støy om tilgjengelig.  
**Første praktiske steg:** Ring sted og be om roligere bord / hjørne.  
**Når menneskelig hjelp bør anbefales:** Angst som lammer; isolasjon over tid — trygg henvisning til fastlege/psykisk helse.

**AI-seedspørsmål (3):**

1. Hvordan forberede meg til støy på restaurant?  
2. Hva kan jeg si til vertskap uten å be om «spesialbehandling»?  
3. Hvilke høreapparatprogram hjelper mest i folkemengde?

**RAG-/kildebehov:** Akustikk og kommunikasjonstips; ikke erstatte behandling for angst.  
**Visuelt anker:** «Før / under / etter» med tre korte punkter per fase.

## Kandidat 11 — P3

| | |
|---|---|
| **Anbefalt arbeidstittel** | Fra skuff til daglig bruk: ett lite løfte du klarer |
| **Alternativ tittel** | Når høreapparatet blir liggende i skuffen |
| **Hvorfor anbefalt** | **Restart-narrativ** uten skam; alternativ navngir vanlig oppgitt problem. |

**Hub:** Restart og mestring  
**Artikkeltype:** Spoke (motivasjon + mikroplan)

**Brukersituasjon:** Har ikke brukt apparatet på dagevis/uker; dårlig samvittighet.  
**Problem brukeren prøver å løse:** Komme tilbake uten «full reset»-press.  
**Hva brukeren skal forstå:** Små sesjoner bygger vane; audiograf er vant til pauser.  
**Hva brukeren skal gjøre:** Sette **ett** daglig vindu (kort), booke støtte hvis nødvendig.  
**Første praktiske steg:** Finn apparat frem, sjekk batteri/lad, bruk 15 min hjemme.  
**Når menneskelig hjelp bør anbefales:** Depresjonsymptomer, hørsel plutselig endret markant.

**AI-seedspørsmål (3):**

1. Hvordan starte med høreapparat igjen etter en pause?  
2. Hva gjør jeg hvis jeg føler meg mislykket med bruken?  
3. Når bør jeg ta kontakt med audiograf etter lang pause?

**RAG-/kildebehov:** Adferdsendring / vaner (lett vektet); ikke terapi i artikkelen.  
**Visuelt anker:** Enkel «uke 1»-plan med tre avmerkingsbokser.

## Kandidat 12 — P3

| | |
|---|---|
| **Anbefalt arbeidstittel** | Audiograf: når det haster, og hva du kan forberede |
| **Alternativ tittel** | Når bør du kontakte audiograf? |
| **Hvorfor anbefalt** | **Prioritering** (haster vs. kan vente) + forberedelse; «Når …»-ALT er klassisk FAQ. |

**Hub:** Bedre lyd i hverdagen  
**Artikkeltype:** Explainer + beslutningsstøtte

**Brukersituasjon:** Usikker på om problem krever time eller kan vente.  
**Problem brukeren prøver å løse:** Unngå unødig angst **og** unødig forsinkelse.  
**Hva brukeren skal forstå:** Akutte tegn vs. alt som kan planlegges.  
**Hva brukeren skal gjøre:** Lage kort notat (når, hvor, hva, hvilke grep prøvd).  
**Første praktiske steg:** Ring / bruk pasientportal etter praksisens system.  
**Når menneskelig hjelp bør anbefales:** Akutt hørselstap, sterke smerter, infeksjonstegn (generell helseveiledning, ikke diagnose i artikkelen).

**AI-seedspørsmål (3):**

1. Hva regnes som akutt hørselproblem jeg må sjekke raskt?  
2. Hva bør jeg ha klart før time hos audiograf?  
3. Kan jeg få tilpasset om apparatet piper litt tidvis?

**RAG-/kildebehov:** Nasjonale helseanbefalinger om akutt hørselstap (haster); generell pasientinformasjon.  
**Visuelt anker:** To kolonner «ring nå / book time» med 3 punkter hver.
