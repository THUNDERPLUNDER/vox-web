# Viddel Article Brief v0.1

## Status

Canonical v0.1 for hvordan Viddel-artikler og spokes skal planlegges, skrives og vurderes i MVP.

Dette dokumentet gjelder før vi skriver nye artikler, rydder eksisterende R1–R3, bygger nye spokes eller redesigner hub/forside basert på innhold.

## Formål

Viddel-artikler skal hjelpe brukeren å forstå en situasjon, gjøre ett konkret neste steg og vite når det er riktig å be om mer hjelp.

Artiklene skal ikke forklare Viddel, teste maler eller vise intern produktlogikk. De skal være nyttige tekster for mennesker som står i en faktisk hørselsrelatert situasjon.

## Grunnregel

Start med brukerens behov, ikke med avsenderens forklaring.

En god Viddel-artikkel svarer tidlig på:

1. Hva skjer?
2. Hva kan jeg gjøre nå?
3. Når bør jeg be om hjelp?

## Skriveregler

1. Start med brukerens behov, ikke avsenderens bakgrunn.
2. Skriv aktivt, konkret og handlingsrettet.
3. Bruk du-form når det gjør teksten tydeligere og mer direkte.
4. Sett viktigste poeng tidlig i setningen, avsnittet og seksjonen.
5. La hver tekstblokk ha én tydelig jobb.
6. Skriv for handling og forståelse, ikke for arkiv.
7. Bruk konkrete ord framfor abstrakte begreper.
8. Kutt pynt, fyll, gjentakelser og interne forklaringer.
9. Gjør lenker og knapper presise: teksten skal si hva brukeren får eller gjør.
10. Skriv redaksjonelt og troverdig, ikke reklamepreget.
11. Skill mellom hovedflyt og fordypning. Legg detaljer, kilder og forbehold der de ikke forstyrrer handlingen.
12. Test hver seksjon mot:
    - Hvilket behov svarer dette på?
    - Hva skal brukeren forstå eller gjøre?
    - Er neste steg tydelig?
13. Skill tydelig mellom trygghet og handling. Teksten kan normalisere og senke terskelen, men hvert avsnitt bør fortsatt hjelpe brukeren videre.

## Artikkelens minimumsstruktur

Hver ny spoke bør ha:

1. Brukersituasjon
2. Kort svar først
3. Hvorfor dette skjer
4. Hva du kan prøve nå
5. Når du bør be om hjelp
6. Neste steg / Article AI-seedspørsmål
7. Eventuell fordypning eller kilder

Før en artikkel skrives, fyll ut:

```text

```
